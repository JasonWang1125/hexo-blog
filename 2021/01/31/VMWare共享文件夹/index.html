VMware 共享文件夹

发表于 2021-01-31 分类于 Software Operation ， VMware Workstation Pro

测试环境

  • VMWare ：15.0
  • 宿主机：Windows10
  • Linux 客户机：Ubuntu 20.04 Desktop 内核版本：5.8

Ubuntu中命令出现权限不够的问题时，在命令前面加上 sudo

为虚拟机启用共享文件夹

  1. 选择该虚拟机，然后选择虚拟机 > 设置。

  2. 在选项选项卡中，选择共享文件夹。

  3. 单击添加以添加共享文件夹。

  1. 按照向导进行选项设置。

具体参数参考 VMWare 官网 / 为虚拟机启用共享文件夹

Linux 客户机 挂载 VMWare 共享文件夹

​ 挂载前要求客户机已经安装了 vmware-tools

  1. 新建挂载目录
  1. 挂载 共享文件夹

挂载命令对应的 Linux 内核版本为 4.0 及以上，具体参数细节和其他内核版本的命令

参考 VMWare 官网 / 在 Linux 客户机中装载共享文件夹

Linux 客户机 解除挂载的 VMWare 共享文件夹

​ 直接删除会报错，即使是 root 也会显示权限不够。同时更致命的是，共享文件夹中的 数据会丢失。

正确解除的方式如下：

  1. 解除挂载
  1. 删除 Linux 中的文件夹
